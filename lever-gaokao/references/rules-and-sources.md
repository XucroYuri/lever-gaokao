## 需要先确认的规则问题

1. 省份和年份：不同省份每年可能调整批次、计划、志愿数量和投档细节。
2. 考试模式：
   - `3+3`：通常不分传统文理，按选考科目限制专业。
   - `3+1+2`：首选物理/历史，再选科目影响专业范围。
   - 传统文理：仍按文科/理科或相关批次规则处理。
3. 志愿单位：
   - 院校专业组。
   - 专业+院校。
   - 院校志愿下填专业。
   - 专项计划、提前批、艺术体育等特殊单位。
4. 投档方式：
   - 平行志愿。
   - 顺序志愿。
   - 一轮投档。
   - 多轮或征集志愿。
5. 调剂范围：
   - 同一院校专业组内调剂。
   - 同一院校内调剂。
   - 不设调剂或按专业+院校直接投档。
6. 批次与特殊类型：
   - 本科提前批。
   - 普通本科批。
   - 国家专项、地方专项、高校专项。
   - 公费师范、优师、军警、航海、民航、医学定向等。
7. 限制条件：
   - 选科。
   - 体检。
   - 单科成绩。
   - 外语语种。
   - 性别或身高。
   - 政审、面试、体测。

## 必查官方资料

每次正式分析应尽量获取：

- 省级考试院当年招生政策文件。
- 高考一分一段表。
- 批次控制线。
- 当年招生计划书或官方计划查询。
- 目标高校招生章程。
- 目标高校当年分省分专业招生计划。
- 近年投档线和专业录取分。
- 征集志愿公告和补录信息。

## 当年实时刷新步骤

当用户说“刚出分”“今年填报”“已经出线”“投档线出了没”“现在能不能报”或要求最终志愿表时，先做当年实时刷新，再继续分析。

1. 确认当前阶段：出分前、出分后、填报期、投档期、征集期或录取后。
2. 按阶段检索省级考试院和官方志愿填报系统，判断省控线、一分一段、招生计划、投档线、征集计划是否已经发布。
3. 对目标高校检索本科招生网，判断招生章程、分省分专业计划、专业组、录取进程、投档/录取统计是否更新。
4. 记录每个资料的发布时间、更新日期、适用批次、适用科类/选科和附件名称。
5. 已发布的当年官方资料优先替换历史数据；未发布的资料写入待核验，不用历史数据冒充今年结果。
6. 如果资料刚发布但页面、附件、第三方转载之间口径冲突，先用省级考试院和高校官方原文；仍冲突时降置信度。

当年刷新结果至少覆盖：

| 数据项 | 主要官方来源 | 未发布时怎么处理 |
| --- | --- | --- |
| 省控线 | 省级考试院 | 只能判断规则预案，不能正式划批次边界 |
| 一分一段 | 省级考试院 | 只可按分数粗筛，必须提醒位次缺失 |
| 招生计划/计划变更 | 省级考试院、官方计划查询、高校本科招生网 | 候选池只能称为高召回，不称全量 |
| 招生章程 | 高校本科招生网、阳光高考 | 章程限制未知时，专业组和退档风险待核验 |
| 院校专业组/分省分专业计划 | 省级计划书、官方计划查询、高校本科招生网 | 不建议输出最终志愿排序 |
| 投档线/最低位次 | 省级考试院 | 只用近年位次模拟，不能写成当年可达结论 |
| 征集志愿 | 省级考试院 | 不提前假设某校一定征集 |
| 专业录取分/录取统计 | 高校本科招生网、省级考试院 | 只作复盘或下一年参考 |

## 历史数据使用方法

历史数据只能用于估算，不得当作当年结果。

优先看：

- 位次，而不是只看分数。
- 近 2-3 年趋势，而不是单年最低分。
- 招生计划增减。
- 专业组内容变化。
- 院校更名、合并、转设、校区变化。
- 热门专业热度变化。
- 本省考生对地域和专业偏好的变化。

需要提醒：

- 新高考改革初期，旧文理数据可比性下降。
- 院校专业组改变会使历史最低分失真。
- 专业+院校模式下，专业热度影响更直接。
- 大幅扩招或缩招会改变位次边界。

## 常见省份机制差异

输出时不要死记某省规则，应当按当年文件核验。可按以下类别理解：

### 院校专业组模式

重点核验：

- 每个专业组包含哪些专业。
- 专业组内是否服从调剂。
- 不可接受专业比例。
- 同校不同专业组分差。

### 专业+院校模式

重点核验：

- 每个志愿是否直接对应一个专业。
- 是否存在调剂。
- 专业热度对录取位次的影响。
- 志愿数量是否足以覆盖安全带。

### 传统院校志愿模式

重点核验：

- 院校内专业志愿数。
- 是否服从专业调剂。
- 专业录取规则：分数优先、专业优先、专业级差等。
- 退档风险。

### 顺序志愿或特殊批次

重点核验：

- 第一志愿权重。
- 面试、体检、政审、体测、单科或性别要求。
- 提前批录取是否影响后续批次。
- 不服从调剂的退档后果。

## 输出要求

在报告中固定列出：

- 本次使用的是哪种规则机制。
- 已核验的官方来源。
- 尚未核验但会影响结论的项目。
- 需要在正式填报前刷新的资料。
